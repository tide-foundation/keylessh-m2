# Gateway Static Assets

These files are served from the KeyleSSH domain for the browser-based RDP client and WebRTC upgrade. They're copies of what the punchd-bridge gateway embeds, but served directly from KeyleSSH for faster loading (no WebSocket relay needed for static files).

## Files

```
gateway/
├── rdp.html               # RDP client page
├── js/
│   ├── rdp-client.js       # RDP client (IronRDP WASM + WebRTC + recording)
│   ├── webrtc-upgrade.js   # WebRTC DataChannel upgrade for endpoints
│   ├── sw.js               # Service Worker for DataChannel HTTP routing
│   └── tide-e2e.js         # End-to-end encryption helper
└── wasm/
    ├── ironrdp_web.js      # IronRDP WASM loader (generated by wasm-pack)
    └── ironrdp_web_bg.wasm # IronRDP WASM binary (~4.3 MB)
```

## Updating IronRDP WASM

The WASM files are built from the [IronRDP](https://github.com/Devolutions/IronRDP) project's `ironrdp-web` crate. KeyleSSH uses a fork at `/home/sasha/IronRDP`.

### Prerequisites

```bash
# Install wasm-pack
cargo install wasm-pack

# Or via npm
npm install -g wasm-pack
```

### Build

```bash
cd /path/to/IronRDP/crates/ironrdp-web

# Build for web target
wasm-pack build --target web --release

# Output is in pkg/
ls pkg/
# ironrdp_web.js
# ironrdp_web_bg.wasm
# ironrdp_web.d.ts
# package.json
```

### Copy to KeyleSSH

```bash
# Copy the two files needed
cp pkg/ironrdp_web.js      /path/to/keylessh/client/public/gateway/wasm/
cp pkg/ironrdp_web_bg.wasm /path/to/keylessh/client/public/gateway/wasm/

# Also update the gateway's embedded copy (used when serving through relay)
cp pkg/ironrdp_web.js      /path/to/keylessh/bridges/punchd-bridge-rs/public/wasm/
cp pkg/ironrdp_web_bg.wasm /path/to/keylessh/bridges/punchd-bridge-rs/public/wasm/
```

### One-liner

```bash
IRONRDP=~/IronRDP
KEYLESSH=~/keylessh

cd $IRONRDP/crates/ironrdp-web && wasm-pack build --target web --release && \
cp pkg/ironrdp_web.js pkg/ironrdp_web_bg.wasm $KEYLESSH/client/public/gateway/wasm/ && \
cp pkg/ironrdp_web.js pkg/ironrdp_web_bg.wasm $KEYLESSH/bridges/punchd-bridge-rs/public/wasm/ && \
echo "WASM updated in both locations"
```

### After Updating

1. **KeyleSSH**: Run `npm run build` to include the new WASM in the build output
2. **Gateway**: Rebuild the gateway binary (`cargo build --release`) — the WASM is embedded via `include_bytes!`
3. **Deploy**: Redeploy KeyleSSH and distribute the new gateway binary

### Verify

Check the WASM version by opening the browser console during an RDP session:
```
[RDP] IronRDP WASM loaded
```

The WASM file size should be ~4-5 MB. If it's significantly larger or smaller, something may be wrong with the build.

## Updating JS Files

The JS files (`rdp-client.js`, `webrtc-upgrade.js`, `sw.js`, `tide-e2e.js`) are maintained in two locations:

1. **KeyleSSH copy**: `client/public/gateway/js/` — served from KeyleSSH domain
2. **Gateway copy**: `bridges/punchd-bridge-rs/public/js/` — embedded in the gateway binary

After editing, sync both copies:

```bash
# If you edited the KeyleSSH copy:
cp client/public/gateway/js/*.js bridges/punchd-bridge-rs/public/js/

# If you edited the gateway copy:
cp bridges/punchd-bridge-rs/public/js/*.js client/public/gateway/js/

# Then rebuild both:
npm run build                           # KeyleSSH
cargo build --release --bin punchd-bridge-rs  # Gateway
```

## RDP HTML Page

The `rdp.html` page also exists in two locations:
- `client/public/gateway/rdp.html` — served from KeyleSSH (references `/gateway/js/rdp-client.js`)
- `bridges/punchd-bridge-rs/public/rdp.html` — embedded in gateway (references `/js/rdp-client.js`)

The KeyleSSH version has the script path prefixed with `/gateway/`. If you update the HTML, keep the script paths consistent for each location.
